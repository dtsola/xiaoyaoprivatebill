# 后端重构方案 - 阶段 1

## 目标
将 `app.py` (3000+ 行) 拆分为模块化结构，**API 路由保留在 app.py 中**。

## 目录结构

```
backend/
├── app.py                  # 主入口 + API 路由 (~500 行)
├── config.py               # 配置常量 (~50 行)
├── requirements.txt
├── data/
│   └── sample_data.csv
├── utils/                  # 工具函数 (新)
│   ├── __init__.py
│   ├── session.py          # 会话管理
│   ├── file_utils.py       # 文件操作
│   └── validators.py       # 数据验证
├── services/               # 业务逻辑 (新)
│   ├── __init__.py
│   ├── data_loader.py      # 数据加载
│   ├── analysis.py         # 数据分析
│   └── generators.py       # 图表数据生成
└── parsers/                # 文件解析 (新)
    ├── __init__.py
    ├── alipay.py           # 支付宝账单解析
    └── wechat.py           # 微信账单解析
```

## 迁移步骤

1. **创建目录结构**
   ```bash
   mkdir -p backend/utils backend/services backend/parsers
   touch backend/utils/__init__.py
   touch backend/services/__init__.py
   touch backend/parsers/__init__.py
   ```

2. **创建 config.py** - 提取所有常量

3. **创建 utils/** - 提取工具函数
   - `session.py` - 会话管理
   - `file_utils.py` - 文件操作
   - `validators.py` - 数据验证

4. **创建 parsers/** - 提取文件解析逻辑
   - `alipay.py` - 支付宝解析
   - `wechat.py` - 微信解析

5. **创建 services/** - 提取业务逻辑
   - `data_loader.py` - 数据加载 (调用 parsers)
   - `analysis.py` - 分析函数
   - `generators.py` - 图表数据生成

6. **重构 app.py** - 保留 API 路由，导入模块

7. **测试验证** - 确保 API 功能正常

## 预期结果

| 文件 | 行数 | 说明 |
|-----|------|-----|
| app.py | ~500 | 仅保留路由和配置 |
| config.py | ~50 | 配置常量 |
| utils/session.py | ~150 | 会话管理 |
| utils/file_utils.py | ~100 | 文件操作 |
| utils/validators.py | ~50 | 数据验证 |
| parsers/alipay.py | ~150 | 支付宝解析 |
| parsers/wechat.py | ~200 | 微信解析 |
| services/data_loader.py | ~200 | 数据加载 |
| services/analysis.py | ~800 | 分析函数 |
| services/generators.py | ~500 | 图表生成 |

**总计**: ~2700 行，分布在 10 个文件中，每个文件职责单一。

## 优点

1. **职责清晰**: 每个模块只负责一件事
2. **易于维护**: 修改分析逻辑不需要动路由
3. **易于测试**: 可以单独测试每个模块
4. **代码复用**: 其他项目可以复用 parser 和 service
5. **协作友好**: 多人可以并行开发不同模块
