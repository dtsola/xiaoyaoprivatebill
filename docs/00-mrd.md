# 精简MRD：小遥账单助手

## 1. 核心问题（一句话）
**解决什么问题？**
帮助用户从支付宝、微信等平台的消费记录中快速了解自己的财务状况，获得个性化的财务规划建议，同时保护用户隐私（数据完全本地处理）。

---

## 2. 目标用户（具体画像）

### 人群特征
- **广泛人群**：不限制特定年龄段或职业
- **有基本记账需求**：使用支付宝/微信进行日常消费
- **隐私意识强**：不希望将财务数据上传到第三方服务器
- **技术门槛低**：期望操作简单，无需复杂的配置

### 痛点场景
1. **账单分散**：消费记录分散在多个平台，难以统一查看
2. **手动记账累**：传统记账需要手动输入，难以坚持
3. **隐私担忧**：现有记账工具多需要上传数据，担心泄露
4. **缺乏分析**：只有流水记录，没有深入的消费分析和建议
5. **不知钱去哪了**：月度结余时才发现超支，缺乏实时感知

### 当前替代方案
| 方案 | 优点 | 缺点 |
|------|------|------|
| 纯手动记账 | 灵活、隐私可控 | 耗时、难以坚持、易遗忘 |
| 各平台自带的账本 | 数据自动同步 | 平台割裂、分析浅、部分收费 |
| 第三方记账APP | 功能丰富 | **需上传数据、隐私风险、广告多** |
| Excel表格 | 灵活、隐私可控 | **需要手动整理、分析门槛高** |

**我们的优势**：数据完全本地处理 + 自动解析账单 + 深度分析

---

## 3. 解决方案（核心功能）

**MVP三个核心功能：**

### 1. 账单解析与导入
- 支持上传支付宝/微信账单文件（CSV/Excel）
- 自动识别和解析账单格式
- 本地处理，数据不上传任何服务器
- 支持多账单合并导入

### 2. 可视化分析看板
- **收支概览**：月度/年度收入、支出、结余
- **消费分类**：按类别（餐饮、交通、购物等）自动分类统计
- **趋势分析**：时间维度消费趋势图
- **异常提醒**：识别异常高额消费

### 3. 智能财务建议
- **支出结构分析**：哪些类别占比过高
- **月度预算建议**：基于历史数据的预算规划
- **储蓄目标建议**：如何优化支出以增加储蓄
- **消费习惯洞察**：识别高频消费场景

## 4. 竞品情况

| 竞品 | 优势 | 劣势 | 差异化机会 |
|------|------|------|-----------|
| 记账城市 | 界面美观、社区活跃 | **需上传数据**、广告多 | **纯本地处理** |
| 钱迹 | 极简设计 | **需上传数据**、高级功能收费 | **完全免费** |
| 各平台自带账本 | 数据自动同步 | 平台割裂、分析浅 | **跨平台整合** |
| Excel/GSheet | 完全灵活、隐私可控 | 需手动整理、技术门槛高 | **自动化解析** |

**核心差异化**：
1. **隐私第一**：数据完全本地处理，不上传任何服务器
2. **跨平台整合**：一个工具搞定支付宝+微信
3. **免费开源**：作为引流工具，完全免费使用

## 6. 风险评估

### 技术风险
- **账单格式变化**：支付宝/微信可能改变账单格式 → 需要持续维护解析逻辑
- **数据量处理**：用户账单可能非常大（数年数据）→ 需要优化前端性能
- **浏览器兼容性**：纯前端方案需要考虑旧浏览器 → 现代浏览器即可

### 市场风险
- **竞争激烈**：记账工具市场已非常成熟 → 靠"隐私保护"差异化切入
- **用户习惯**：用户可能已有常用记账工具 → 转换成本高
- **留存问题**：账单分析属于低频需求 → 需要思考如何提高使用频率

### 时间风险
- 开发周期：预计2-4周完成MVP
- 维护成本：账单格式变化需要跟进
