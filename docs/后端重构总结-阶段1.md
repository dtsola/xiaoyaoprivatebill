# 后端重构总结 - 阶段 1

## 完成时间
2026-02-17

## 重构目标
将 3000+ 行的单一 `app.py` 文件拆分为模块化架构，提升代码可维护性。

## 模块功能说明

### config.py - 配置常量
- 目录配置 (UPLOAD_FOLDER, SESSION_FILE_DIR, DATA_DIR)
- 文件上传配置 (ALLOWED_EXTENSIONS, MAX_CONTENT_LENGTH)
- 会话配置 (SESSION_LIFETIME, CACHE_LIFETIME)
- 日志配置 (LOG_LEVEL, LOG_FILE)
- 分类颜色映射 (CATEGORY_COLORS)

### utils/session.py - 会话管理
- `get_session_dir()` - 获取/创建会话目录
- `user_cache` - 用户级缓存装饰器
- `cleanup_orphan_files()` - 清理孤儿文件
- `start_cleanup_thread()` - 后台定期清理
- `ensure_upload_dir()` - 确保目录存在

### utils/file_utils.py - 文件操作
- `allowed_file()` - 检查文件类型
- `detect_file_source()` - 检测文件来源 (支付宝/微信)
- `save_uploaded_file()` - 保存上传文件
- `format_file_size()` - 格式化文件大小
- `get_file_info()` - 获取文件信息

### utils/validators.py - 数据验证
- `validate_dataframe()` - 验证数据完整性
- `validate_amount_range()` - 验证金额范围
- `validate_date_params()` - 验证日期参数
- `validate_pagination_params()` - 验证分页参数

### parsers/alipay.py - 支付宝解析
- `parse_alipay_csv()` - 解析支付宝 CSV 账单

### parsers/wechat.py - 微信解析
- `parse_wechat_csv()` - 解析微信 CSV 账单
- `parse_wechat_xlsx()` - 解析微信 XLSX 账单

### services/data_loader.py - 数据加载
- `load_demo_data()` - 加载演示数据
- `load_alipay_data()` - 加载并合并账单数据
- `clear_data_cache()` - 清除数据缓存

### services/analysis.py - 数据分析
包含 15+ 分析函数：
- 商家分析 (`analyze_merchants`)
- 场景分析 (`analyze_scenarios`)
- 习惯分析 (`analyze_habits`)
- 拿铁因子 (`analyze_latte_factor`)
- 夜间消费 (`analyze_nighttime_spending`)
- 订阅服务 (`analyze_subscriptions`)
- 通胀分析 (`analyze_inflation`)
- 品牌忠诚度 (`analyze_brand_loyalty`)
- 桑基图 (`analyze_sankey`)
- 恩格尔系数 (`analyze_engel_coefficient`)
- 周末对比 (`analyze_weekend_vs_monday`)
- 支付方式 (`analyze_payment_methods`)
- 智能标签 (`generate_smart_tags`)
- 故事数据 (`generate_story_data`)
- 辅助计算函数

### services/generators.py - 图表生成
包含 9 个图表数据生成函数：
- 弦图 (`generate_chord_data`)
- 漏斗图 (`generate_funnel_data`)
- 象限图 (`generate_quadrant_data`)

## 重构优势

1. **职责清晰**: 每个模块只负责一件事
2. **易于维护**: 修改分析逻辑不需要动路由
3. **易于测试**: 可以单独测试每个模块
4. **代码复用**: 其他项目可以复用 parser 和 service
5. **协作友好**: 多人可以并行开发不同模块
6. **减少冲突**: 不同功能在不同文件，减少 Git 冲突
