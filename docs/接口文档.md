# 老账单项目 API 接口文档

> 源文件: `backend/BK/app.py` (备份版本)

## 一、API 概览

| 类别 | 接口数量 | 说明 |
|-----|---------|-----|
| 页面路由 | 10 | 前端页面路由 (已废弃) |
| 数据分析 API | 12 | 核心数据分析接口 |
| 文件管理 API | 5 | 上传、删除、列出文件 |
| 会话管理 API | 4 | 演示模式、会话状态 |
| 元数据 API | 3 | 获取可用日期、年份、分类 |

## 三、数据分析 API

**请求**:
```
GET /api/analysis?year=2024&min_amount=10&max_amount=1000
```

**参数**:
| 参数 | 类型 | 必填 | 说明 |
|-----|------|-----|-----|
| year | int | 否 | 筛选年份 |
| min_amount | float | 否 | 最小金额 |
| max_amount | float | 否 | 最大金额 |

**响应**:
```json
{
    "success": true,
    "data": {
        "merchant_analysis": { ... },       // 商家分析
        "scenario_analysis": { ... },       // 消费场景分析
        "habit_analysis": { ... },          // 消费习惯分析
        "latte_factor": { ... },            // 拿铁因子分析
        "nighttime_analysis": { ... },      // 夜间消费分析
        "subscription_analysis": { ... },   // 订阅服务分析

        "engel_coefficient": { ... },       // 恩格尔系数
        "weekend_monday": { ... },          // 周末vs周一对比

        "tags": [ ... ],                    // 智能标签
        "payment_analysis": { ... },        // 支付方式分析

        "funnel_data": { ... },             // 漏斗图数据
```

### 3.2 月度分析 `/api/monthly_analysis`

**请求**:
```
GET /api/monthly_analysis?year=2024&month=6&min_amount=10&max_amount=1000
```

**参数**:
| 参数 | 类型 | 必填 | 说明 |
|-----|------|-----|-----|
| year | int | 是 | 年份 |
| month | int | 是 | 月份 (1-12) |
| min_amount | float | 否 | 最小金额 |
| max_amount | float | 否 | 最大金额 |

**响应**:
```json
{
    "success": true,
    "data": {
        "stats": {
            "balance": 5200.50,              // 结余
            "total_expense": 3500.00,        // 总支出
            "total_income": 8700.50,         // 总收入
            "expense_count": 45,             // 支出笔数
            "income_count": 3,               // 收入笔数
            "comparisons": {                 // 与上月对比
                "balance": { "change": 300.00, "rate": 6.12 },
                "expense": { "change": -200.00, "rate": -5.41 },
                "income": { "change": 500.00, "rate": 6.10 }
            }
        },
        "daily_data": {                      // 每日数据
            "expense": { "dates": [...], "amounts": [...] },
            "income": { "dates": [...], "amounts": [...] }
        },
        "categories": {                      // 分类统计
            "expense": { "names": [...], "amounts": [...] },
            "income": { "names": [...], "amounts": [...] }
        },
        "categories_source": {               // 按来源的分类统计
            "expense": [{ "来源": "支付宝", "交易分类": "餐饮", "金额": 500 }],
            "income": [...]
        }
    }
}
```

### 3.4 交易记录 `/api/transactions`

**请求**:
```
GET /api/transactions?page=1&per_page=20&year=2024&month=6&type=支出&category=餐饮美食&min_amount=10&search=美团
```

**参数**:
| 参数 | 类型 | 必填 | 说明 |
|-----|------|-----|-----|
| page | int | 否 | 页码 (默认 1) |
| per_page | int | 否 | 每页条数 (默认 20) |
| year | int | 否 | 年份筛选 |
| month | int | 否 | 月份筛选 |
| date | str | 否 | 日期筛选 (YYYY-MM-DD) |
| hour | int | 否 | 小时筛选 (0-23) |
| category | str | 否 | 分类筛选 |
| min_amount | float | 否 | 最小金额 |
| max_amount | float | 否 | 最大金额 |
| type | str | 否 | 收支类型 (收入/支出) |
| search | str | 否 | 搜索关键词 (商品/对方/分类) |

**响应**:
```json
{
    "success": true,
    "transactions": [
        {
            "time": "2024-06-15 12:30:00",
            "description": "美团外卖",
            "category": "餐饮美食",
            "type": "支出",
            "amount": 35.50,
            "status": "交易成功",
            "counterparty": "美团点评"
        }
    ],
    "pagination": {
        "current_page": 1,
        "per_page": 20,
        "total_pages": 15,
        "total_records": 285
    }
}
```

## 四、文件管理 API

### 4.1 上传文件 `/api/upload`

**请求**:
```
POST /api/upload
Content-Type: multipart/form-data
```

**参数**:
| 参数 | 类型 | 必填 | 说明 |
|-----|------|-----|-----|
| file | File | 是 | 支付宝 CSV 或微信 CSV/XLSX |

**响应**:
```json
{
    "success": true,
    "filename": "alipay_2024.csv",
    "message": "文件上传成功"
}
```

**支持格式**:
- 支付宝 CSV (GBK 编码)
- 微信 CSV (UTF-8-sig 编码)
- 微信 XLSX

## 七、关键业务流程

### 7.1 文件上传流程

```
用户上传文件
    │
    ├─→ 检查文件类型 (CSV/XLSX)
    │
    ├─→ 获取/创建会话目录 (/tmp/flask_uploads/{user_id}/)
    │
    ├─→ 生成安全文件名 (secure_filename)
    │
    ├─→ 检测文件来源 (支付宝/微信)
    │
    ├─→ 保存文件
    │
    ├─→ 更新会话时间戳
    │
    └─→ 清除数据缓存
```

---

### 7.2 数据加载流程 (`load_alipay_data()`)

```
load_alipay_data()
    │
    ├─→ [演示模式]
    │    └─→ 加载 sample_data.csv
    │
    └─→ [正常模式]
         ├─→ 遍历会话目录下所有文件
         │
         ├─→ [CSV 文件]
         │    ├─→ 检测来源 (支付宝/微信)
         │    ├─→ 支付宝: GBK 编码, 查找表头行
         │    └─→ 微信: UTF-8-sig 编码, 列名映射
         │
         ├─→ [XLSX 文件]
         │    └─→ header=16, 列名映射
         │
         ├─→ 统一数据处理
         │    ├─→ 添加 '月份', '日期' 列
         │    ├─→ 标记退款 (是否退款)
         │    └─→ 添加 '来源' 标识
         │
         └─→ 合并所有数据, 按时间排序
```

---

### 7.3 会话管理流程

```
用户首次访问
    │
    ├─→ get_session_dir()
    │    ├─→ 生成 user_id (token_hex(16))
    │    ├─→ 记录 created_at 时间戳
    │    └─→ 创建会话目录
    │
    ├─→ 会话有效期检查 (30 分钟)
    │    ├─→ 超时 → 清理旧目录
    │    └─→ 重新生成会话
    │
    └─→ 数据缓存 (5 分钟)
         └─→ @user_cache 装饰器
```

---

### 7.4 演示模式流程

```
POST /api/demo/enter
    │
    ├─→ session['is_demo'] = True
    │
    ├─→ session['user_id'] = 'demo_user'
    │
    └─→ 所有 API 调用返回 sample_data.csv 数据

POST /api/demo/exit
    │
    ├─→ 清除 is_demo 标记
    │
    └─→ 恢复正常模式
```

---

### 7.5 数据分析流程 (以 `/api/analysis` 为例)

```
GET /api/analysis
    │
    ├─→ load_alipay_data()
    │    └─→ 获取合并后的 DataFrame
    │
    ├─→ 应用筛选条件
    │    ├─→ 年份筛选
    │    └─→ 金额范围筛选
    │
    ├─→ 执行各类分析函数
    │    ├─→ analyze_merchants()       # 商家分析
    │    ├─→ analyze_scenarios()       # 场景分析
    │    ├─→ analyze_habits()          # 习惯分析
    │    ├─→ analyze_latte_factor()    # 拿铁因子
    │    ├─→ analyze_nighttime_spending()  # 夜间消费
    │    ├─→ analyze_subscriptions()   # 订阅分析
    │    ├─→ analyze_inflation()       # 通胀分析
    │    ├─→ analyze_brand_loyalty()   # 品牌忠诚度
    │    ├─→ analyze_sankey()          # 桑基图
    │    ├─→ analyze_engel_coefficient()  # 恩格尔系数
    │    ├─→ analyze_weekend_vs_monday()  # 周末对比
    │    ├─→ analyze_payment_methods()    # 支付方式
    │    ├─→ generate_story_data()     # 故事数据
    │    ├─→ generate_smart_tags()     # 智能标签
    │    ├─→ generate_chord_data()     # 弦图
    │    ├─→ generate_funnel_data()    # 漏斗图
    │    ├─→ generate_quadrant_data()  # 象限图
    │    ├─→ generate_radar_data()     # 雷达图
    │    ├─→ generate_wordcloud_data() # 词云
    │    ├─→ generate_themeriver_data() # 主题河流
    │    ├─→ generate_boxplot_data()   # 箱线图
    │    ├─→ generate_heatmap_data()   # 热力图
    │    └─→ generate_pareto_data()    # 帕累托图
    │
    └─→ 返回综合分析结果
```

## 九、配置说明

### 9.1 Flask 配置

| 配置项 | 值 | 说明 |
|-------|-----|-----|
| SESSION_COOKIE_SECURE | False | 本地开发环境 |
| SESSION_COOKIE_HTTPONLY | True | 防止 JS 访问 cookie |
| SESSION_COOKIE_SAMESITE | 'Lax' | 防止 CSRF |
| PERMANENT_SESSION_LIFETIME | 30 分钟 | 会话过期时间 |
| MAX_CONTENT_LENGTH | 16MB | 最大上传文件 |
| UPLOAD_FOLDER | /tmp/flask_uploads | 临时文件目录 |

### 9.2 环境变量

| 变量 | 说明 |
|-----|-----|
| FLASK_SECRET_KEY | Flask 会话密钥 |
| NEW_FRONTEND | 设为 '1' 启用新前端 |

## 十、错误处理

所有 API 返回格式:

**成功**:
```json
{
    "success": true,
    "data": { ... }
}
```

**失败**:
```json
{
    "success": false,
    "error": "错误信息"
}
```

常见错误:
- `没有文件被上传` - 未选择文件
- `不支持的文件类型` - 非 CSV/XLSX
- `未找到任何支付宝(.csv)或微信(.xlsx)账单文件` - 无有效数据
- `数据缺少必需列` - 数据格式错误

---

## 十一、文件解析规则

### 支付宝 CSV
- **编码**: GBK (尝试失败则 UTF-8)
- **表头行**: 自动查找包含 "交易时间" 的行
- **状态行**: 查找包含 "交易状态" 的行
- **退款标识**: "退款成功", "交易关闭"

### 微信 CSV
- **编码**: UTF-8-sig
- **表头行**: 自动查找包含 "交易时间" 和 "交易类型" 的行
- **列名映射**:
  - 交易类型 → 交易分类
  - 商品 → 商品说明
  - 金额(元) → 金额
  - 当前状态 → 交易状态
  - 支付方式 → 收/付款方式

### 微信 XLSX
- **表头行**: 第 17 行 (固定)
- **列名映射**: 同微信 CSV
