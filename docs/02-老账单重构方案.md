# 老账单项目重构方案 - 前端优先渐进式分离

## 一、方案原则

**核心策略**: 先分离前端，后端保持不变，测试通过后再重构后端

### 优势
1. **风险可控**: 前端独立开发，不影响现有后端功能
2. **并行开发**: 新前端开发的同时，老系统继续可用
3. **渐进切换**: 通过配置开关实现新旧前端平滑切换
4. **后端兼容**: 保持现有 API 完全兼容，零修改

### 实施路径
```
┌─────────────────────────────────────────────────────────────┐
│  阶段1: 前端独立开发   │  后端: 保持不变，继续服务  │
├─────────────────────────────────────────────────────────────┤
│  阶段2: 双前端共存     │  新前端 │ 老前端  │  共用后端 │
├─────────────────────────────────────────────────────────────┤
│  阶段3: 切换验证       │  验证新前端功能完整性         │
├─────────────────────────────────────────────────────────────┤
│  阶段4: 后端重构       │  前端稳定后，再重构后端       │
└─────────────────────────────────────────────────────────────┘
```

## 二、阶段 1: 前端独立开发 (2-3周)

### 1.1 后端改动 - 最小化 (1天)

**只需要添加一个静态文件服务配置**:

```python
# olds/app.py (在现有代码基础上添加)

# 添加这一段，支持新前端的开发模式
import os
from flask import send_from_directory

# 新前端开发模式: 当环境变量 NEW_FRONTEND=1 时，返回新前端
@app.route('/')
@app.route('/<path:path>')
def serve_new_frontend(path=''):
    # 如果开启了新前端模式
    if os.environ.get('NEW_FRONTEND') == '1':
        frontend_dist = os.path.join(os.path.dirname(__file__), '..', 'frontend', 'dist')
        if os.path.exists(frontend_dist):
            if path and os.path.exists(os.path.join(frontend_dist, path)):
                return send_from_directory(frontend_dist, path)
            return send_from_directory(frontend_dist, 'index.html')

    # 否则使用原有逻辑
    if path == '':
        return render_template('index.html')
    # ... 原有路由逻辑
```

### 1.4 API 客户端 - 复用现有 API

**文件**: `frontend/src/api/client.js`

```javascript
// API 基础配置
const API_BASE = import.meta.env.VITE_API_BASE_URL || '/api';

// 复用现有的所有 API 端点，零修改后端
export const api = {
    // 数据分析
    getYearlyAnalysis: (params) => get('/yearly_analysis', params),
    getMonthlyAnalysis: (params) => get('/monthly_analysis', params),
    getCategoryExpenses: (params) => get('/category_expenses', params),
    getTransactions: (params) => get('/transactions', params),
    getAnalysis: (params) => get('/analysis', params),
    getTimeAnalysis: (params) => get('/time_analysis', params),
    getSummary: (params) => get('/summary', params),

    // 元数据
    getAvailableDates: () => get('/available_dates'),
    getAvailableYears: () => get('/available_years'),
    getCategories: () => get('/categories'),

    // 文件管理
    uploadFile: (formData) => post('/upload', formData),  // 注意: 使用 multipart/form-data
    getFiles: () => get('/files'),
    deleteFile: (filename) => delete_(`/files/${filename}`),
    clearData: () => post('/clear_data'),

    // 会话管理
    enterDemo: () => post('/demo/enter'),
    exitDemo: () => post('/demo/exit'),
    getSessionStatus: () => get('/session/status'),
    getTimeRemaining: () => get('/session/time_remaining'),
};

// 请求封装
function get(endpoint, params) {
    const url = new URL(API_BASE + endpoint, window.location.origin);
    if (params) {
        Object.entries(params).forEach(([k, v]) => url.searchParams.set(k, v));
    }
    return fetch(url)
        .then(r => r.json())
        .then(d => {
            if (!d.success) throw new Error(d.error || '请求失败');
            return d.data;
        });
}

function post(endpoint, data) {
    const isFormData = data instanceof FormData;
    return fetch(API_BASE + endpoint, {
        method: 'POST',
        headers: isFormData ? {} : { 'Content-Type': 'application/json' },
        body: isFormData ? data : JSON.stringify(data)
    }).then(r => r.json()).then(d => d.success ? d.data : Promise.reject(d.error));
}

function delete_(endpoint) {
    return fetch(API_BASE + endpoint, { method: 'DELETE' })
        .then(r => r.json())
        .then(d => d.success ? d.data : Promise.reject(d.error));
}
```

## 三、阶段 2: 双前端共存 (1周)

### 3.1 新老前端切换机制

**运行方式**:

```bash
# 方式1: 使用老前端 (现有)
cd olds
python app.py
# 访问: http://localhost:8080

# 方式2: 使用新前端
cd frontend
npm run dev
# 访问: http://localhost:3000 (开发)
# 或构建后通过 Flask 服务:
cd frontend && npm run build
cd olds && NEW_FRONTEND=1 python app.py
```

## 八、风险与应对

| 风险 | 应对措施 |
|-----|---------|
| 后端 API 不兼容 | **零修改后端**，复用现有 API |
