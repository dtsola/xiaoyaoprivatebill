## 2. Docker 一键部署（推荐）

### 2.1 前置要求

| 软件 | 版本要求 | 说明 |
|------|---------|------|
| **Docker** | 20.10+ | 容器运行时 |
| **Docker Compose** | 2.0+ | 容器编排工具 |

**安装 Docker**：
```bash
# Ubuntu/Debian
curl -fsSL https://get.docker.com | sh

# macOS
brew install --cask docker

# Windows
# 下载 Docker Desktop: https://www.docker.com/products/docker-desktop
```

### 2.2 快速启动

```bash
# 1. 克隆项目
git clone https://github.com/dtsola/xiaoyaoprivatebill.git
cd xiaoyaoprivatebill

# 2. 一键启动（后台运行）
docker-compose up -d

# 3. 查看启动日志
docker-compose logs -f

# 4. 访问应用
# 浏览器打开: http://localhost:8080
```

### 2.3 常用命令

```bash
# 启动服务
docker-compose up -d

# 停止服务
docker-compose down

# 重启服务
docker-compose restart

# 查看日志
docker-compose logs -f          # 所有服务
docker-compose logs -f backend  # 仅后端
docker-compose logs -f frontend # 仅前端

# 查看服务状态
docker-compose ps

# 进入容器
docker-compose exec backend bash
docker-compose exec frontend sh

# 重新构建镜像
docker-compose build

# 清理所有数据（包括上传的文件）
docker-compose down -v
```

### 2.4 环境变量配置

在 [docker-compose.yml](../docker-compose.yml) 中可配置以下环境变量：

| 变量名 | 默认值 | 说明 |
|--------|--------|------|
| `SECRET_KEY` | 自动生成 | Flask 会话密钥 |
| `SESSION_LIFETIME` | 1800 | 会话有效期（秒） |
| `MAX_CONTENT_LENGTH` | 16777216 | 上传文件大小限制（字节） |
| `UPLOAD_FOLDER` | /app/data | 上传文件存储目录 |

### 2.5 数据持久化

```bash
# 查看数据卷
docker volume ls

# 备份数据卷
docker run --rm -v xiaoyaoprivatebill_backend-data:/data -v $(pwd):/backup alpine tar czf /backup/data-backup.tar.gz /data

# 恢复数据卷
docker run --rm -v xiaoyaoprivatebill_backend-data:/data -v $(pwd):/backup alpine tar xzf /backup/data-backup.tar.gz
```

## 3. 手动部署

### 3.1 系统要求

| 配置项 | 最低配置 | 推荐配置 |
|--------|---------|---------|
| **CPU** | 2 核 | 4 核+ |
| **内存** | 2GB | 4GB+ |
| **硬盘** | 10GB | 20GB+ |
| **操作系统** | Linux/macOS/Windows | Ubuntu 22.04 LTS |

### 3.2 后端部署

#### 安装 Python 环境

```bash
# Ubuntu/Debian
sudo apt update
sudo apt install python3.10 python3.10-venv python3-pip

# macOS
brew install python@3.10

# Windows
# 下载 Python 3.10: https://www.python.org/downloads/
```

#### 部署后端

```bash
# 1. 进入后端目录
cd backend

# 2. 创建虚拟环境（Windows）
py -3.10 -m venv venv
venv\Scripts\activate

# 2. 创建虚拟环境（Linux/macOS）
python3.10 -m venv venv
source venv/bin/activate

# 3. 安装依赖
pip install -r requirements.txt

# 4. 配置环境变量（可选）
export FLASK_SECRET_KEY="your-secret-key"
export SESSION_LIFETIME=1800

# 5. 启动服务
python app.py
# 后端运行在: http://localhost:5000
```

#### 使用进程管理工具（可选）

**使用 Supervisor**：

```bash
# 安装 Supervisor
sudo apt install supervisor

# 创建配置文件
sudo nano /etc/supervisor/conf.d/xiaoyao-backend.conf
```

```ini
[program:xiaoyao-backend]
directory=/path/to/backend
command=/path/to/backend/venv/bin/python app.py
autostart=true
autorestart=true
stderr_logfile=/var/log/xiaoyao-backend.err.log
stdout_logfile=/var/log/xiaoyao-backend.out.log
user=www-data
```

```bash
# 重载配置
sudo supervisorctl reread
sudo supervisorctl update

# 启动服务
sudo supervisorctl start xiaoyao-backend
```

### 3.4 Nginx 配置

#### 安装 Nginx

```bash
# Ubuntu/Debian
sudo apt install nginx

# macOS
brew install nginx

# Windows
# 下载 Nginx: http://nginx.org/en/download.html
```

#### 配置站点

创建配置文件 `/etc/nginx/sites-available/xiaoyao-bill`：

```nginx
# 小遥私人账单分析助手 - Nginx 配置

server {
    listen 80;
    server_name localhost;

    # 前端静态文件
    root /path/to/frontend/dist;
    index index.html;

    # Gzip 压缩
    gzip on;
    gzip_vary on;
    gzip_min_length 1024;
    gzip_types text/plain text/css text/xml text/javascript application/x-javascript application/xml+rss application/json application/javascript;

    # 静态资源缓存
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }

    # API 代理到后端
    location /api {
        proxy_pass http://127.0.0.1:5000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # 上传文件大小限制
        client_max_body_size 16M;

        # 超时配置
        proxy_connect_timeout 60s;
        proxy_send_timeout 60s;
        proxy_read_timeout 60s;
    }

    # Vue Router history 模式支持
    location / {
        try_files $uri $uri/ /index.html;
    }

    # 禁止访问隐藏文件
    location ~ /\. {
        deny all;
    }

    # 日志配置
    access_log /var/log/nginx/xiaoyao-bill-access.log;
    error_log /var/log/nginx/xiaoyao-bill-error.log;
}
```

#### 启用配置

```bash
# 创建软链接
sudo ln -s /etc/nginx/sites-available/xiaoyao-bill /etc/nginx/sites-enabled/

# 测试配置
sudo nginx -t

# 重载 Nginx
sudo systemctl reload nginx

# 启动 Nginx
sudo systemctl start nginx

# 设置开机自启
sudo systemctl enable nginx
```

## 4. 常见问题

### 4.1 Docker 部署问题

**Q: docker-compose 启动失败**

A: 检查端口占用
```bash
# 检查 8080 端口
lsof -i :8080  # Linux/macOS
netstat -ano | findstr :8080  # Windows

# 修改端口
# 编辑 docker-compose.yml，将 8080 改为其他端口
```

**Q: 容器启动后无法访问**

A: 查看容器日志
```bash
docker-compose logs -f
```

**Q: 后端健康检查失败**

A: 检查后端是否正常启动
```bash
docker-compose exec backend python -c "import socket; s = socket.socket(); s.connect(('localhost', 5000)); s.close()"
```

### 4.2 手动部署问题

**Q: 后端启动失败（端口占用）**

A: 修改后端端口
```python
# 编辑 backend/config.py
PORT = 5001  # 改为其他端口
```

同时修改 Nginx 配置中的 `proxy_pass`。

**Q: 前端构建失败**

A: 清除缓存重试
```bash
cd frontend
rm -rf node_modules dist
npm install
npm run build
```

**Q: API 请求 404**

A: 检查 Nginx 配置
```bash
# 确认 API 代理配置正确
sudo nginx -t

# 查看 Nginx 错误日志
sudo tail -f /var/log/nginx/xiaoyao-bill-error.log
```

**Q: 上传文件失败**

A: 检查文件大小限制
```nginx
# Nginx 配置
client_max_body_size 16M;

# Flask 配置 (backend/config.py)
MAX_CONTENT_LENGTH = 16 * 1024 * 1024  # 16MB
```

### 4.3 数据处理问题

**Q: 账单解析失败**

A: 确认文件格式
- 支付宝：CSV 格式，UTF-8 编码
- 微信：CSV 或 XLSX 格式

**Q: 数据分析超时**

A: 优化配置
```python
# backend/config.py
# 增加会话超时时间
SESSION_LIFETIME_MINUTES = 120
```

## 6. 开发环境端口配置总结

| 环境 | 前端地址 | 后端地址 | API 代理 |
|------|---------|---------|---------|
| **Docker 生产** | localhost:8080 | backend:5000 | Nginx Proxy |
| **手动生产** | localhost:80 (Nginx) | localhost:5000 | Nginx Proxy |
| **本地开发** | localhost:3000 | localhost:5000 | Vite Proxy |

**关键说明**：
- Docker 生产环境：前端 Nginx 监听 8080，`/api/*` 代理到容器内 `backend:5000`
- 手动生产环境：前端 Nginx 监听 80，`/api/*` 代理到本地 `127.0.0.1:5000`
- 本地开发环境：Vite Dev Server 监听 3000，`/api/*` 代理到本地 `localhost:5000`
