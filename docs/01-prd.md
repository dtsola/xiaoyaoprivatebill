# PRD：小遥私人账单分析助手 MVP版本

## 1. 产品定位（一句话）

为 使用支付宝/微信进行日常消费的广大人群 提供 数据完全本地处理、跨平台整合分析 的 账单分析工具

---

## 2. 用户故事（核心场景）

**场景1：首次用户导入账单**
- 作为 一个关注隐私的用户
- 我想要 上传支付宝/微信账单文件
- 以便于 快速了解我的消费情况，同时确保数据不泄露

**场景2：查看财务趋势**
- 作为 一个想要改善财务状况的用户
- 我想要 查看月度和年度消费趋势
- 以便于 发现消费模式，制定更好的预算计划

**场景3：深度消费分析**
- 作为 一个想要详细了解消费习惯的用户
- 我想要 查看分类分析和消费洞察
- 以便于 了解钱花在哪里，识别可优化的支出

**场景4：隐私顾虑用户**
- 作为 一个对隐私敏感的用户
- 我想要 确认数据完全在本地处理
- 以便于 放心使用而不担心财务数据泄露

**场景5：演示体验**
- 作为 一个新用户
- 我想要 先体验演示模式了解功能
- 以便于 决定是否上传我的真实账单数据

## 3. 功能清单（MVP范围）

### P0功能（必须有）

#### 核心数据功能
- [ ] **账单解析与导入**：支持上传支付宝(CSV)和微信(XLSX/CSV)账单文件
- [ ] **多文件合并**：支持同时上传多个账单文件并合并分析
- [ ] **数据自动识别**：自动识别账单来源（支付宝/微信）
- [ ] **数据标准化**：将不同平台的账单格式统一处理
- [ ] **会话管理**：基于会话的数据处理，30分钟超时自动清理

#### 核心分析功能
- [ ] **年度分析**：年度消费趋势、收支比例、分类占比
- [ ] **月度分析**：月度消费明细、环比变化、趋势图
- [ ] **分类分析**：按消费类别的统计、趋势、占比分析
- [ ] **时间分析**：每日热力图、时段分布、工作日/周末对比
- [ ] **消费洞察**：消费画像、商家分析、习惯识别

#### 交易查询功能
- [ ] **交易记录列表**：完整的交易明细展示
- [ ] **搜索功能**：按商品说明/交易对方搜索
- [ ] **筛选功能**：按金额范围、分类、时间筛选
- [ ] **分页展示**：大量交易数据的分页加载

#### 演示模式
- [ ] **演示数据生成**：内置示例账单数据
- [ ] **演示模式切换**：支持进入/退出演示模式
- [ ] **演示标识**：清晰显示当前为演示模式

#### 隐私保护
- [ ] **本地数据处理**：所有分析在服务器端完成，不永久存储
- [ ] **会话自动清理**：会话结束后自动删除所有上传文件
- [ ] **隐私声明**：明确告知用户数据处理方式

### P1功能（最好有）

#### 高级分析
- [ ] **拿铁因子分析**：识别小额高频支出
- [ ] **深夜消费分析**：分析夜间消费模式
- [ ] **隐形订阅检测**：识别周期性订阅支出
- [ ] **恩格尔系数**：食品支出占比分析
- [ ] **品牌忠诚度分析**：最常光顾商家统计
- [ ] **RFM分析**：最近消费、频率、金额分析

#### 数据管理
- [ ] **文件列表**：显示已上传的账单文件
- [ ] **删除单个文件**：删除指定的账单文件
- [ ] **清空所有数据**：一键清除所有上传数据
- [ ] **数据导出**：导出分析后的交易数据

#### 可视化增强
- [ ] **桑基图**：资金流向可视化
- [ ] **词云图**：消费关键词云
- [ ] **雷达图**：多维度消费画像
- [ ] **帕累托图**：80/20支出分析

#### 用户体验
- [ ] **会话倒计时**：显示剩余会话时间
- [ ] **自动过期提醒**：会话即将过期时提醒用户
- [ ] **图表导出**：保存分析图表为图片

### P2功能（未来考虑）

#### 个性化定制
- [ ] **自定义分类**：用户可创建和编辑消费分类
- [ ] **分类规则**：设置自动分类规则
- [ ] **预算管理**：设置分类预算和提醒

#### 高级功能
- [ ] **AI理财建议**：基于消费数据的个性化建议
- [ ] **目标追踪**：储蓄目标进度追踪
- [ ] **账单对比**：不同时期消费对比
- [ ] **账单合并导入**：支持拖拽批量导入

#### 平台扩展
- [ ] **云闪付支持**：支持银联云闪付账单
- [ ] **京东支付支持**：支持京东支付账单
- [ ] **美团支付支持**：支持美团支付账单

## 4. 核心流程图

```
用户访问首页
    ↓
选择【上传账单】或【演示模式】
    ↓
    ├─ 演示模式 → 加载示例数据 → 直接查看分析
    │
    └─ 上传账单 → 选择文件 → 上传 → 账单解析
            ↓
        数据清洗和标准化
            ↓
        主界面（年度分析页）
            ↓
        用户选择分析维度
            ↓
    ┌───────┼───────┐
    ↓       ↓       ↓       ↓
年度   月度   分类   时间   洞察   交易
分析   分析   分析   分析   分析   记录
    ↓
    查看详细数据和可视化图表
    ↓
    获取消费洞察和建议
    ↓
    会话结束（30分钟）→ 数据自动清理
```

## 6. 数据埋点计划

| 事件 | 触发时机 | 关键参数 | 用途 |
|------|---------|---------|------|
| page_view | 页面访问 | page_name, source | 了解用户使用路径 |
| file_upload_attempt | 尝试上传文件 | file_type, file_size | 分析用户数据源 |
| file_upload_success | 文件上传成功 | file_type, records_count | 追踪导入成功率 |
| file_upload_fail | 文件上传失败 | error_code, file_type | 优化导入体验 |
| demo_enter | 进入演示模式 | - | 评估演示效果 |
| demo_exit | 退出演示模式 | duration_in_demo | 了解演示留存 |
| analysis_view | 查看分析页面 | analysis_type, time_range | 了解热门功能 |
| chart_interact | 图表交互 | chart_type, interaction | 优化可视化 |
| filter_apply | 应用筛选 | filter_type, filter_value | 了解用户偏好 |
| search_execute | 执行搜索 | search_term, results | 优化搜索功能 |
| export_data | 导出数据 | export_type, record_count | 了解数据需求 |
| session_timeout | 会话超时 | session_duration | 优化会话时长 |

## 7. 非功能需求

### 性能要求
- **首屏加载**：< 3秒（不含演示数据）
- **账单解析**：< 5秒（1000条记录）
- **图表渲染**：< 1秒（100个数据点）
- **搜索响应**：< 500ms
- **文件上传**：支持最大 16MB 文件

### 兼容性
- **浏览器**：Chrome 90+、Safari 14+、Edge 90+
- **移动端**：iOS Safari、Android Chrome
- **分辨率**：支持 320px - 4K 显示器
- **文件编码**：支持 UTF-8、GBK（支付宝CSV）

### 安全与隐私
- **数据存储**：仅临时存储，会话结束后自动删除
- **数据传输**：HTTPS 加密传输
- **文件验证**：严格验证文件类型和大小
- **会话管理**：30分钟超时，自动清理
- **隐私声明**：明确告知用户数据处理方式

### 可用性
- **错误处理**：友好的错误提示和引导
- **空状态设计**：无数据时的引导页面
- **加载状态**：清晰的数据加载反馈
- **响应式设计**：适配桌面和移动设备
- **无障碍支持**：键盘导航、语义化HTML

### 可维护性
- **代码规范**：遵循项目代码规范
- **注释文档**：核心功能添加注释
- **日志记录**：记录关键操作和错误
- **监控告警**：异常情况的监控和告警

## 附录：API端点清单

### 文件管理
- `POST /api/upload` - 上传账单文件
- `GET /api/files` - 列出已上传文件
- `DELETE /api/files/<filename>` - 删除指定文件
- `POST /api/clear_data` - 清空所有数据
- `POST /api/cleanup` - 清理临时文件

### 数据分析
- `GET /api/summary` - 总体财务摘要
- `GET /api/yearly_analysis` - 年度分析
- `GET /api/monthly_analysis` - 月度分析
- `GET /api/category_analysis` - 分类分析
- `GET /api/time_analysis` - 时间模式分析
- `GET /api/transactions` - 交易数据（分页+筛选）

### 会话管理
- `GET /api/session/status` - 会话状态
- `GET /api/session/time_remaining` - 剩余时间
- `POST /api/demo/enter` - 进入演示模式
- `POST /api/demo/exit` - 退出演示模式
