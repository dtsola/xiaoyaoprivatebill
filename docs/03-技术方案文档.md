## 1. 技术选型

### 1.1 前端

| 技术 | 版本 | 用途 | 选择理由 |
|------|------|------|---------|
| **Vue** | 3.4+ | 渐进式前端框架 | 学习曲线平缓，生态成熟，适合中小型项目 |
| **Vite** | 5.0+ | 构建工具 | 开发服务器启动快速，HMR 热更新效率高 |
| **Vue Router** | 4.2+ | 路由管理 | Vue 官方路由解决方案，与框架深度集成 |
| **Pinia** | 2.1+ | 状态管理 | Vue 3 官方推荐，类型友好，API 简洁 |
| **ECharts** | 5.4.3 | 数据可视化 | 功能强大，图表类型丰富，复用现有代码 |
| **Fetch API** | - | HTTP 客户端 | 原生 API，无需额外依赖 |

### 1.2 后端

| 技术 | 版本 | 用途 | 选择理由 |
|------|------|------|---------|
| **Python** | 3.10+ | 后端开发语言 | 数据处理能力强，生态成熟 |
| **Flask** | 2.0+ | Web 框架 | 轻量级，灵活，适合小型项目和快速开发 |
| **Pandas** | Latest | 数据处理核心 | 强大的数据分析和处理能力，适合账单数据解析 |
| **NumPy** | Latest | 数值计算 | 高效的数值运算，Pandas 依赖 |
| **OpenPyXL** | Latest | Excel 文件处理 | 支持 .xlsx 格式读取，微信账单支持 |

### 1.3 部署

| 组件 | 部署方式 | 说明 |
|------|---------|------|
| **前端** | 本地静态文件服务 | 隐私优先，数据不上传 |
| **后端** | 本地 Flask 服务 | 用户电脑本地运行 |
| **数据存储** | 本地临时文件 | 用户上传的账单文件临时存储 |

> **注意**：本项目采用**本地优先**架构，所有数据在用户本地处理，不上传到任何服务器。

## 4. API 接口设计

### 4.1 数据分析接口

#### GET /api/available_dates
获取可用的年份数据

**响应**：
```json
{
  "success": true,
  "data": {
    "years": [2022, 2023, 2024],
    "date_range": {
      "start": "2022-01-01",
      "end": "2024-12-31"
    }
  }
}
```

#### GET /api/available_years
获取可用年份列表

**响应**：
```json
{
  "success": true,
  "data": [2022, 2023, 2024]
}
```

#### GET /api/categories
获取消费分类列表

**响应**：
```json
{
  "success": true,
  "data": ["餐饮美食", "交通出行", "购物消费", ...]
}
```

#### GET /api/yearly_analysis
获取年度分析数据

**请求参数**：
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| year | int | 是 | 年份，如 2024 |

**响应**：见 3.3 节年度分析响应

#### GET /api/monthly_analysis
获取月度分析数据

**请求参数**：
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| year | int | 是 | 年份 |
| month | int | 是 | 月份 (1-12) |

**响应**：见 3.3 节月度分析响应

#### GET /api/category_expenses
获取分类消费数据

**请求参数**：
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| year | int | 是 | 年份 |
| category | string | 否 | 分类名称 |

**响应**：
```json
{
  "success": true,
  "data": {
    "categories": [
      {"name": "餐饮美食", "amount": 12000.00, "count": 245},
      {"name": "交通出行", "amount": 8000.00, "count": 120}
    ]
  }
}
```

#### GET /api/time_analysis
获取时间维度分析数据

**请求参数**：
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| year | int | 是 | 年份 |
| type | string | 否 | 分析类型 (hourly/dayly/weekly) |

**响应**：
```json
{
  "success": true,
  "data": {
    "hourly_distribution": [
      {"hour": 0, "amount": 120.50},
      {"hour": 1, "amount": 45.00}
    ],
    "daily_distribution": [...],
    "weekly_distribution": [...]
  }
}
```

#### GET /api/analysis
获取综合分析数据

**请求参数**：
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| start_date | string | 是 | 开始日期 YYYY-MM-DD |
| end_date | string | 是 | 结束日期 YYYY-MM-DD |

**响应**：
```json
{
  "success": true,
  "data": {
    "summary": {...},
    "trends": {...},
    "insights": [...]
  }
}
```

#### GET /api/summary
获取数据概要

**响应**：
```json
{
  "success": true,
  "data": {
    "total_transactions": 3456,
    "date_range": {...},
    "total_income": 150000.00,
    "total_expense": 85000.00
  }
}
```

## 6. 技术风险

| 风险 | 影响 | 概率 | 应对方案 |
|------|------|------|---------|
| **大文件解析性能问题** | 高 | 中 | 使用 Pandas 分块读取，添加进度反馈 |
| **内存占用过高** | 中 | 中 | 限制单次上传文件大小，及时释放缓存 |
| **浏览器兼容性** | 低 | 低 | 使用现代浏览器特性，提供兼容性提示 |
| **账单格式变化** | 高 | 中 | 解析器模块化设计，便于扩展适配 |
| **前端图表渲染性能** | 中 | 低 | 大数据量时采用数据抽样/分页 |
| **会话超时数据丢失** | 中 | 中 | 提供数据导出功能，会话到期前提醒 |
